# Cleaner V0.2 UI / Flow 设计文档

日期：2026-06-12

版本：V0.2

状态：待用户评审

关联 PRD：`docs/superpowers/specs/2026-06-12-c-drive-cleaner-v0.2-design.zh-CN.md`

## 1. 设计目标

V0.2 的界面目标是让普通用户低成本完成两件事：

1. 找出并清理重复文件。
2. 找出并安全迁移大文件。

界面参考 WPS 清理大师的工具型结构：左侧工具栏、右侧任务页、大按钮启动、结果页顶部空间摘要。但 Cleaner 必须保持自己的品牌和安全表达，不复制 WPS 的素材、VIP 入口、图标和文案。

V0.2 的体验关键词：

- 一眼知道能释放多少空间。
- 默认以 C 盘释放为核心。
- 用户可以主动处理其他盘符。
- 推荐勾选要明确、可取消、可追溯。
- 删除和迁移都必须有确认页。
- 所有高影响操作都要可恢复或可解释。

## 2. 主框架

### 2.1 左侧工具栏

左侧固定宽度工具栏，采用 Cleaner 扁平 C Logo。

导航项：

1. C 盘清理
2. 重复文件清理
3. 大文件迁移
4. 隐私清理 `V0.3`

底部入口：

- 清理历史
- 设置

交互规则：

- 当前模块高亮。
- 隐私清理可见但不可执行，点击后显示“将在 V0.3 提供”说明。
- C 盘清理沿用 V0.1 流程。
- 重复文件清理和大文件迁移进入 V0.2 新流程。
- 切换模块前，如果当前有未完成选择或执行中任务，必须弹出确认。

### 2.2 右侧内容区

每个工具页保持统一布局：

- 顶部：模块图标、标题、副标题、主操作按钮。
- 中部：设置区或结果区。
- 底部：辅助入口、历史、保护路径或下一步按钮。

按钮层级：

- 主按钮：开始查找、清理已选、开始迁移。
- 次按钮：返回、清空选择、更改位置。
- 危险按钮不使用红色大按钮，避免误点；危险确认放在确认页里完成。

## 3. 重复文件清理

### 3.1 页面 1：查找设置页

目的：让用户选择文件类型和扫描位置。

顶部区域：

- 标题：重复文件清理
- 副标题：快速查找重复文件，释放磁盘空间
- 主按钮：开始查找

文件类型：

- 图片
- 文档
- 音频
- 视频
- 压缩包
- 自定义后缀

默认选择：

- 图片、文档、压缩包默认选中。
- 音频、视频默认不选中，避免首次扫描耗时过长。
- 用户可以一键全选。

自定义后缀：

- 输入框占位：`请输入后缀，多个用逗号分隔，如 jpg, mp4, zip`
- 输入后实时校验。
- 非法后缀给出内联提示，不阻止其他类型扫描。

查找位置：

- 默认勾选 C 盘。
- 展示本机其他盘符。
- 支持添加文件夹。
- 支持移除用户添加的文件夹。

辅助入口：

- 高级选项
- 保护路径
- 清理历史

高级选项：

- 是否显示疑似重复。
- 最小文件大小。
- 是否跳过系统目录和程序目录。

### 3.2 页面 2：扫描中

目的：让用户知道扫描进度和当前状态。

显示内容：

- 大号百分比。
- 当前扫描位置。
- 当前文件类型。
- 已扫描文件数。
- 已发现严格重复组数量。
- 已发现疑似重复数量。

进度阶段：

1. 枚举文件。
2. 按大小初筛。
3. 计算 Hash。
4. 生成重复组。

交互：

- 支持取消扫描。
- 取消后可选择返回设置页。
- 如果已经有部分结果，V0.2 默认不展示部分结果，避免用户误判完整性。

### 3.3 页面 3：结果页

目的：展示重复组，并让用户用安全策略快速选择。

顶部摘要：

- 可释放总空间。
- 已选空间。
- C 盘可释放。
- 其他位置可释放。
- 严格重复组数量。
- 疑似重复组数量。

结果分区：

1. 严格重复
2. 疑似重复

严格重复组卡片：

- 组标题：重复文件组、文件数量、可释放空间。
- 组标签：跨盘重复 / C 盘内重复 / 其他盘重复。
- 组操作：
  - 展开 / 收起
  - 智能选择本组
  - 清空本组选择
  - 添加保护路径

组内文件行：

- checkbox
- 文件名
- 简化位置
- 所在盘符
- 文件大小
- 修改时间
- 推荐状态：保留 / 清理
- 保护路径标记

疑似重复组：

- 默认折叠。
- 不参与“一键清理”。
- 不默认勾选。
- 文案明确：疑似重复仅供参考，需要手动判断。

智能勾选策略：

- 优先清理 C 盘副本。
- 保留最新。
- 保留最旧。
- 保留指定目录。
- 手动选择。

默认策略：

- 跨 C 盘和其他盘：优先删除 C 盘副本。
- 同盘重复：保留最新。
- 其他盘重复：可推荐，但标注“非 C 盘释放”。

保护规则：

- 每组至少保留 1 个文件。
- 保护路径内文件不能自动勾选。
- 用户手动选择保护路径内文件，需要二次确认。

### 3.4 页面 4：清理确认页

目的：在删除前让用户明确风险和恢复方式。

确认内容：

- 将清理的文件数量。
- 涉及的重复组数量。
- 预计释放空间。
- C 盘预计释放空间。
- 其他位置预计释放空间。

必须说明：

- 文件将移入回收站。
- 每个重复组至少保留 1 份。
- 保护路径不会被自动清理。
- 回收站失败时不会永久删除。

确认控件：

- checkbox：我确认将以上文件移入回收站。
- 主按钮：确认清理已选文件。
- 次按钮：返回修改。

### 3.5 页面 5：清理中

显示内容：

- 大号百分比。
- 当前处理文件序号。
- 已移入回收站数量。
- 已跳过数量。
- 已失败数量。

执行策略：

- 单个文件失败不阻断其他文件。
- 每个文件删除前重新检查：
  - 文件仍存在。
  - 文件仍属于扫描得到的重复组。
  - 该组仍保留至少 1 份。
  - 不在保护路径内。

### 3.6 页面 6：清理结果页

结果摘要：

- 实际释放空间。
- C 盘释放空间。
- 其他位置释放空间。
- 成功数量。
- 跳过数量。
- 失败数量。

列表：

- 成功项：默认折叠。
- 跳过项：展示原因。
- 失败项：展示原因和建议。

可执行动作：

- 查看清理历史。
- 再次扫描。
- 返回重复文件清理首页。

## 4. 大文件迁移

### 4.1 页面 1：查找设置页

目的：让用户选择扫描位置和大文件阈值。

顶部区域：

- 标题：大文件迁移
- 副标题：定位大文件，迁移到其他位置释放 C 盘空间
- 主按钮：开始查找

磁盘卡片：

- 磁盘名称。
- 已用空间。
- 剩余空间。
- 容量条。
- checkbox。

默认选择：

- C 盘默认选中。
- 其他盘默认不选中。

阈值：

- 100MB+
- 500MB+ 默认
- 1GB+
- 自定义

辅助入口：

- 高级筛选
- 迁移历史
- 保护路径

高级筛选：

- 文件类型。
- 修改时间。
- 是否跳过系统目录。
- 是否跳过程序目录。

### 4.2 页面 2：扫描中

显示内容：

- 大号百分比。
- 当前扫描位置。
- 已扫描文件数。
- 已发现大文件数量。
- 已发现大文件总空间。

阶段：

1. 枚举文件。
2. 按阈值筛选。
3. 识别类型。
4. 排除保护路径和关键目录。

### 4.3 页面 3：结果页

顶部摘要：

- 发现大文件总空间。
- 已选迁移空间。
- 预计释放 C 盘空间。
- 其他位置涉及空间。

分类：

- 视频
- 压缩包
- 安装包
- 镜像文件
- 文档
- 其他大文件

文件行：

- checkbox
- 文件名
- 简化位置
- 所在盘符
- 大小
- 修改时间
- 类型标签
- 推荐状态
- 保护路径标记

推荐规则：

- C 盘用户目录下的大文件可推荐迁移。
- 下载目录、桌面、文档目录中的大文件优先展示。
- 系统目录、程序目录、配置目录不自动推荐。
- 保护路径内文件不自动推荐。

主操作：

- 下一步：设置迁移位置。

### 4.4 页面 4：迁移设置页

目标位置：

- 默认建议：`<目标盘>:\Cleaner_MigratedFiles`
- 用户可以修改为其他文件夹。
- 目标路径必须存在或可创建。

目标校验：

- 目标空间必须足够。
- 目标路径不能等于源路径。
- 目标路径不能是源路径的子目录。
- 目标路径不能在保护路径冲突范围内。

原文件处理：

1. 保留原文件。
2. 复制校验成功后移入回收站。

默认选项：

- 默认选择保留原文件。
- 如果用户目标是释放 C 盘空间，界面提示需要选择“移入回收站”才会释放原位置空间。

确认摘要：

- 将迁移文件数量。
- 迁移总空间。
- 预计释放 C 盘空间。
- 目标位置。
- 原文件处理策略。

### 4.5 页面 5：迁移确认页

确认内容：

- 目标目录。
- 文件数量。
- 总迁移大小。
- 原文件处理策略。
- 复制后校验说明。

必须说明：

- 校验失败不会处理原文件。
- 回收站失败不会永久删除原文件。
- 如果选择保留原文件，迁移不会释放原位置空间。

确认控件：

- checkbox：我确认迁移设置无误。
- 主按钮：开始迁移。
- 次按钮：返回修改。

### 4.6 页面 6：迁移执行页

进度分三段：

1. 复制文件。
2. 校验文件。
3. 处理原文件。

显示内容：

- 总百分比。
- 当前文件名。
- 当前阶段。
- 已复制空间。
- 已校验数量。
- 已释放空间。

失败策略：

- 单个文件失败不阻止其他文件。
- 复制失败：保留原文件。
- 校验失败：保留原文件。
- 回收站失败：目标文件保留，原文件保留。

### 4.7 页面 7：迁移结果页

结果摘要：

- 迁移成功空间。
- 实际释放空间。
- C 盘释放空间。
- 成功数量。
- 失败数量。
- 跳过数量。

结果分区：

- 迁移成功。
- 已释放原位置空间。
- 迁移成功但未释放空间。
- 失败。

可执行动作：

- 查看迁移历史。
- 继续查找大文件。
- 打开目标文件夹。

## 5. 历史页

历史页聚合：

- C 盘清理历史。
- 重复文件清理历史。
- 大文件迁移历史。

列表字段：

- 时间。
- 模块。
- 操作类型。
- 释放空间。
- 迁移空间。
- 成功数量。
- 失败数量。

隐私规则：

- 不显示完整路径。
- 不显示文件名。
- 不保存 Hash 原文。
- 只展示脱敏摘要。

## 6. 设置页

设置项：

- 保护路径管理。
- 默认扫描范围。
- 重复文件默认智能勾选策略。
- 大文件默认阈值。
- 历史保留天数。
- 桌面快捷方式。
- C 盘右键菜单入口。
- 定时扫描提醒。

默认值：

- 默认扫描范围：C 盘。
- 重复文件策略：C 盘优先释放，同盘保留最新。
- 大文件阈值：500MB。
- 历史保留：30 天。
- 桌面快捷方式：关闭。
- C 盘右键菜单：关闭。
- 定时提醒：关闭。

## 7. 状态与空态

### 7.1 空结果

重复文件：

- 文案：未发现可安全清理的重复文件。
- 操作：调整文件类型或扫描位置后重试。

大文件：

- 文案：未发现超过当前阈值的大文件。
- 操作：降低阈值或添加扫描位置。

### 7.2 权限不足

文案：

- 部分位置无法读取，已跳过。
- 不提供权限处理或运行模式切换入口。
- 用户可以选择其他文件夹或关闭占用程序后重试。

### 7.3 保护路径命中

文案：

- 该文件位于保护路径内，默认不处理。
- 如需处理，请先从保护路径中移除或手动二次确认。

## 8. UI 文案约定

避免使用：

- “深度清理”
- “彻底删除”
- “一键粉碎”
- “绕过权限”
- “强制处理”

推荐使用：

- “移入回收站”
- “保留一份”
- “复制并校验”
- “实际释放空间”
- “保护路径”
- “返回修改”

## 9. 验收清单

### 9.1 重复文件

- 查找设置页包含文件类型、扫描位置、自定义后缀。
- 扫描页显示百分比和阶段。
- 结果页区分严格重复和疑似重复。
- 智能勾选至少支持 C 盘优先、保留最新、保留最旧。
- 每个重复组至少保留 1 份。
- 确认页明确删除进入回收站。
- 结果页区分成功、跳过、失败。

### 9.2 大文件迁移

- 查找设置页包含磁盘卡片和阈值。
- 默认阈值为 500MB。
- 结果页按文件类型分组。
- 迁移设置页要求目标位置。
- 用户必须选择原文件处理策略。
- 迁移执行页区分复制、校验、处理原文件。
- 迁移结果页区分迁移空间和实际释放空间。

### 9.3 框架

- 左侧导航包含 V0.2 和 V0.3 模块。
- 历史页不展示完整路径和文件名。
- 设置页包含保护路径。
- 所有高影响操作执行前都有确认页。
