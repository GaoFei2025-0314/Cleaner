# C Drive Cleaner V0.2 中文 PRD / Spec

日期：2026-06-12

版本：V0.2

状态：待用户评审

## 1. 产品定位

V0.2 将 C Drive Cleaner 从单一 C 盘清理向“本地磁盘空间整理工具”扩展。产品继续以释放 C 盘空间为主目标，但允许用户主动选择其他盘符或文件夹参与扫描与清理。

V0.2 学习 WPS 清理大师的功能结构和用户心智：

- 左侧工具栏承载多个清理工具入口。
- 重复文件清理支持文件类型筛选、扫描位置选择、智能勾选。
- 大文件迁移支持选择磁盘或文件夹、查找大文件、选择迁移位置。
- 清理历史和保护路径作为降低误删风险的辅助功能。

V0.2 不复制 WPS 的品牌、视觉素材、VIP 标识、文案和图标。功能层面参考公开资料和用户截图，交互与安全策略按本项目重新设计。

## 3. V0.2 范围

### 3.1 本版本实做

V0.2 实做两个核心模块：

1. 重复文件清理
2. 大文件迁移

同时保留 V0.1 已有的 C 盘清理能力，并调整左侧导航结构，为 V0.3 隐私清理预留入口。

### 3.2 本版本不做

V0.2 不做：

- 浏览器历史、密码、自动填表、剪贴板、注册表痕迹等隐私清理。
- 图片相似度、视频相似度、文档语义相似等“近似重复”自动删除。
- 永久删除文件。
- 自动杀进程。
- 自动修改用户配置文件。
- 云端文件内容分析。
- 上传完整路径、文件名或文件内容。
- 管理员权限、提权清理、系统组件清理。

隐私清理整体放入 V0.3 单独设计。

## 4. 总体设计原则

### 4.1 C 盘优先，但支持其他盘

- C 盘是默认目标和主清理对象。
- 用户可以添加其他盘符或文件夹。
- 扫描结果和释放空间统计必须区分“C 盘可释放”和“其他位置可释放”。
- 推荐策略优先释放 C 盘，但不能隐藏其他盘的处理能力。

### 4.2 默认推荐要保守

- 推荐删除只允许基于严格重复文件。
- 严格重复定义为：文件大小相同，内容 Hash 相同。
- 疑似重复可以展示，但不能默认勾选。
- 所有重复组必须至少保留 1 份文件。
- 删除进入回收站，不做永久删除。

### 4.3 迁移必须可恢复

- 大文件迁移先复制到目标位置。
- 复制完成后必须校验大小和内容 Hash。
- 校验成功后，用户可选择保留原文件或将原文件移入回收站。
- 校验失败不得删除或移动原文件。

### 4.4 历史只保存脱敏摘要

V0.2 本地保存简要历史，用于用户回看处理结果，但不保存完整路径和文件名。

允许保存：

- 操作时间。
- 模块类型：重复文件清理 / 大文件迁移。
- 释放空间或迁移空间。
- 成功、跳过、失败数量。
- 目标盘符统计，如 C 盘释放多少、其他盘释放多少。
- 错误类型汇总，如文件占用、权限不足、回收站失败。

禁止保存：

- 完整路径。
- 文件名。
- 文件内容。
- Hash 原文。
- 用户名。
- 聊天记录、文档内容、图片内容。

## 5. 信息架构

左侧工具栏采用参考软件式结构：

1. C 盘清理
2. 重复文件清理
3. 大文件迁移
4. 隐私清理（V0.3 预留，不可执行或显示“后续版本”）
5. 设置 / 历史（可放在左下角或右上菜单）

V0.2 首页优先进入 C 盘清理或上次使用模块，但导航必须让用户明确看到三个可用模块：

- C 盘清理
- 重复文件清理
- 大文件迁移

## 6. 重复文件清理

### 6.1 入口页

入口页结构参考用户截图和 WPS 功能心智：

- 页面标题：重复文件清理
- 副标题：快速查找重复文件，释放磁盘空间
- 主按钮：开始查找
- 文件类型：
  - 图片
  - 文档
  - 音频
  - 视频
  - 压缩包
  - 自定义后缀
- 查找位置：
  - 默认勾选 C 盘。
  - 用户可以勾选其他盘符。
  - 用户可以添加文件夹。
  - 用户可以移除扫描位置。
- 辅助入口：
  - 高级选项
  - 清理历史
  - 保护路径

### 6.2 文件类型规则

默认类型按 WPS 风格组织：

- 图片：jpg、jpeg、png、gif、bmp、webp、heic、tiff
- 文档：doc、docx、xls、xlsx、ppt、pptx、pdf、txt、md、csv
- 音频：mp3、wav、flac、aac、m4a、ogg
- 视频：mp4、mov、avi、mkv、wmv、flv、webm
- 压缩包：zip、rar、7z、tar、gz、bz2、xz

自定义后缀要求：

- 用户输入逗号或分号分隔的后缀。
- 后缀只允许字母、数字和少量安全字符。
- 不能输入通配符删除规则。
- 自定义后缀只影响扫描候选，不影响删除安全判断。

### 6.3 扫描策略

重复文件扫描分三层：

1. 按文件大小分组。
2. 对同大小文件计算内容 Hash。
3. Hash 相同且大小相同的文件进入严格重复组。

可选展示疑似重复：

- 文件名高度相似。
- 文件大小接近。
- 修改时间接近。

疑似重复只展示为“疑似重复”，不能自动勾选，不能进入一键清理推荐。

### 6.4 智能勾选策略

V0.2 支持智能勾选：

- 保留最新文件。
- 保留最旧文件。
- 优先保留非 C 盘文件，删除 C 盘副本。
- 保留指定目录。
- 手动选择。

默认策略：

1. 如果重复组同时存在 C 盘文件和非 C 盘文件，优先推荐删除 C 盘副本，保留非 C 盘副本。
2. 如果重复组只在 C 盘内，默认保留最新文件，推荐删除较旧副本。
3. 如果重复组只在其他盘内，也可以智能推荐，但界面必须标注为“非 C 盘释放”。
4. 不管策略如何，每组必须至少保留 1 份。

### 6.5 结果页

结果页按重复组展示：

- 组标题：重复文件组、文件数量、可释放空间。
- 组内文件：
  - 文件名可展示，但历史不保存。
  - 所在盘符和简化位置。
  - 大小。
  - 修改时间。
  - 推荐保留 / 推荐清理状态。
  - 是否在保护路径内。
- 顶部汇总：
  - 可释放总空间。
  - 已选空间。
  - C 盘可释放。
  - 其他位置可释放。
- 操作：
  - 智能勾选
  - 清空选择
  - 加入保护路径
  - 清理已选

### 6.6 清理确认

执行前必须出现确认页：

- 列出将清理的重复组数量。
- 显示将移入回收站的文件数量。
- 显示预计释放空间。
- 明确说明每个重复组都会保留至少 1 份。
- 明确说明删除进入回收站。

用户确认后才能执行。

### 6.7 清理执行

执行时要求：

- 删除前重新确认文件仍存在。
- 删除前重新确认文件仍属于扫描得到的 Hash 组。
- 删除前重新确认该组仍会保留至少 1 份。
- 删除前检查保护路径。
- 删除失败不能中断其他文件。
- 优先移入回收站。
- 回收站不可用或失败时，本项失败，不自动永久删除。

## 7. 大文件迁移

### 7.1 入口页

入口页结构参考 WPS 风格：

- 页面标题：大文件迁移
- 副标题：定位大文件，迁移到其他位置释放 C 盘空间
- 主按钮：开始查找
- 查找位置：
  - 默认选择 C 盘。
  - 可选择其他盘符。
  - 可添加文件夹。
- 磁盘卡片：
  - 磁盘名称。
  - 已用空间。
  - 剩余空间。
  - 容量条。
  - 是否选中。
- 辅助入口：
  - 迁移历史
  - 高级筛选

### 7.2 大文件阈值

默认阈值：500MB 起。

提供筛选项：

- 100MB+
- 500MB+（默认）
- 1GB+
- 自定义大小

### 7.3 文件分类

大文件结果按类型分组：

- 视频
- 压缩包
- 安装包
- 镜像文件
- 文档
- 其他大文件

V0.2 不自动推荐迁移系统目录、程序安装目录、配置目录里的关键文件。

### 7.4 迁移目标

迁移目标规则：

- 用户必须选择目标盘符或目标文件夹。
- 默认目标目录建议为：`<目标盘>:\CDriveCleaner_MigratedFiles`
- 目标目录名称不得使用 WPS 品牌名。
- 如果目标目录不存在，软件创建目录。
- 如果目标空间不足，不能开始迁移。
- 如果目标路径与源路径相同或互为父子目录，不能开始迁移。

### 7.5 原文件处理

用户在迁移前必须选择：

1. 保留原文件。
2. 复制校验成功后将原文件移入回收站。

默认选项建议为“保留原文件”，但界面要强调“如果要释放 C 盘空间，请选择移入回收站”。

### 7.6 迁移执行

执行流程：

1. 复制文件到目标目录。
2. 校验目标文件大小。
3. 计算或复用 Hash 校验内容一致。
4. 写入迁移结果。
5. 如果用户选择处理原文件，则将原文件移入回收站。

失败策略：

- 复制失败：保留原文件，删除不完整目标文件或标记失败。
- 校验失败：保留原文件，目标文件标记异常，不处理原文件。
- 回收站失败：目标文件保留，原文件保留，结果显示“迁移成功但未释放原位置空间”。
- 单个文件失败不影响其他文件。

## 8. 保护路径

保护路径用于防止误删重要备份或用户指定目录。

V0.2 要求：

- 用户可以添加保护路径。
- 重复文件清理和大文件迁移都尊重保护路径。
- 保护路径下的文件不能被自动勾选删除。
- 保护路径下的大文件可以展示，但默认不推荐迁移或删除。
- 用户手动选择保护路径内文件时，必须二次确认。

保护路径保存在本地设置中。

## 9. 清理历史与迁移历史

V0.2 保存脱敏摘要历史。

历史列表展示：

- 时间。
- 模块。
- 释放空间或迁移空间。
- 成功数量。
- 失败数量。
- 跳过数量。
- C 盘释放空间。
- 其他位置释放空间。

历史详情不展示完整路径和文件名。需要排错时，只展示错误类型和简化位置，例如“C 盘用户文档目录下文件”。

## 10. 设置

V0.2 设置项：

- 保护路径管理。
- 默认扫描范围。
- 重复文件默认智能勾选策略。
- 大文件默认阈值。
- 清理历史保留天数。
- 是否创建桌面快捷方式。
- 是否添加 C 盘右键菜单入口。
- 是否启用定时扫描提醒。

桌面快捷方式、C 盘右键菜单和定时扫描提醒参考 WPS 功能结构，但默认关闭，由用户主动开启。

## 12. 错误处理

统一错误类型：

- 文件不存在。
- 文件正在使用。
- 权限不足。
- 回收站不可用。
- 目标空间不足。
- 复制失败。
- 校验失败。
- 保护路径阻止。
- 安全检查失败。

用户文案要求：

- 用普通语言解释。
- 默认不显示完整路径。
- 提供可执行建议，例如“关闭相关软件后重试”“更换迁移目标盘”“取消保护路径后重新选择”。

## 13. 验收标准

### 13.1 重复文件清理

- 默认扫描范围包含 C 盘。
- 用户可以添加其他盘符或文件夹。
- 支持图片、文档、音频、视频、压缩包、自定义后缀。
- 严格重复必须基于大小和内容 Hash。
- 疑似重复不能默认勾选。
- 智能勾选必须保留每组至少 1 个文件。
- 跨 C 盘和其他盘重复时，默认优先释放 C 盘。
- 删除必须进入回收站。
- 回收站失败时不能永久删除。
- 保护路径内文件不能被自动勾选。

### 13.2 大文件迁移

- 默认阈值为 500MB。
- 支持 100MB+、500MB+、1GB+、自定义阈值。
- 用户必须选择目标位置。
- 复制后必须校验。
- 校验失败不得处理原文件。
- 用户必须选择原文件处理策略。
- 选择移入回收站时，回收站失败不得永久删除。
- 迁移结果必须区分“迁移空间”和“实际释放空间”。

### 13.3 历史和隐私

- 保存本地脱敏摘要历史。
- 历史不包含完整路径或文件名。
- 历史能展示模块、时间、释放空间、成功/失败/跳过数量。
- 匿名统计仍不得包含路径、文件名、用户名、文件内容。

### 13.4 UI

- 左侧工具栏展示 C 盘清理、重复文件清理、大文件迁移、隐私清理。
- 隐私清理在 V0.2 中不能执行，只能标注后续版本。
- 重复文件入口页具有文件类型、扫描位置、开始查找。
- 大文件迁移入口页具有磁盘卡片、阈值、开始查找。
- 结果页顶部显示可释放空间、已选空间、C 盘释放空间。
- 所有危险动作执行前必须有确认页。

## 14. 后续版本预留

V0.3 方向：

- 隐私清理。
- 浏览器痕迹分类。
- 系统使用痕迹分类。
- 剪贴板和运行记录处理。
- 更严格的隐私权限说明。

V0.4 及以后可考虑：

- 图片相似度查重。
- 视频相似度查重。
- 文件归档建议。
- 更完整的迁移回溯能力。

## 15. 已确认决策记录

- V0.2 范围选择：重复文件清理 + 大文件迁移。
- 隐私清理放到 V0.3。
- 重复文件使用实用清理版：自动推荐副本，但删除进回收站并二次确认。
- 大文件迁移使用安全迁移版：复制、校验、再处理原文件。
- 扫描范围：默认 C 盘，用户可选择其他盘符或文件夹。
- 主导航：参考软件式左侧工具栏。
- 重复文件类型：图片、文档、音频、视频、压缩包、自定义后缀。
- 重复文件准确性：严格重复可推荐，疑似重复只展示。
- 大文件默认阈值：500MB。
- 大文件迁移后原文件处理：用户每次选择。
- 重复文件默认保留策略：C 盘优先释放；同盘默认保留最新。
- 其他盘符：也可智能推荐，但 C 盘优先。
- 历史：本地保存脱敏简要历史。
