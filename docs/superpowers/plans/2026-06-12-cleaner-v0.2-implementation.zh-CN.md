# Cleaner V0.2 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 构建 Cleaner V0.2：在保留 V0.1 C 盘清理的基础上，新增重复文件清理、大文件迁移、设置、保护路径、脱敏历史和左侧工具型导航。

**Architecture:** Rust/Tauri 后端负责所有文件系统扫描、Hash、回收站、复制校验、保护路径、安全复查和 AppData JSON 持久化；React/TypeScript 前端只展示状态、收集用户选择并调用命令。长耗时任务统一走 operation 模型：前端启动任务，后端异步执行并通过 Tauri events 推送真实进度、完成结果和取消状态。

**Tech Stack:** Tauri 2、Rust、Serde、Walkdir、Sysinfo、`sha2`、`trash`、React 18、TypeScript、Vite、Vitest、Testing Library、Cargo tests。

---

## Scope And Defaults

- V0.2 使用一个主实施计划，执行时按任务分批提交；不要一次提交所有改动。
- 采用真实进度和可取消任务：新增 `start_*` 命令、`cancel_operation` 命令和 `cleaner-operation-progress` / `cleaner-operation-finished` 事件。
- 设置、保护路径和脱敏历史保存到 Tauri AppData JSON；历史禁止保存完整路径、文件名、Hash 原文、用户名和内容。
- 文件删除统一改为“移入回收站”；回收站失败时该项失败，不允许永久删除兜底。
- 桌面快捷方式、C 盘右键菜单、定时扫描提醒在 V0.2 设置页展示为默认关闭入口，不执行系统级创建、注册表或计划任务改动。
- 隐私清理只显示 V0.3 预留入口，不执行扫描或清理。

Implement `OperationRegistry` in `src-tauri/src/v2/operations.rs`:

```rust
use std::collections::HashMap;
use std::sync::atomic::AtomicBool;
use std::sync::{Arc, Mutex};

#[derive(Clone)]
pub struct OperationToken {
    pub operation_id: String,
    pub cancelled: Arc<AtomicBool>,
}

#[derive(Default)]
pub struct OperationRegistry {
    operations: Mutex<HashMap<String, Arc<AtomicBool>>>,
}

impl OperationRegistry {
    pub fn register(&self) -> OperationToken {
        let operation_id = uuid::Uuid::new_v4().to_string();
        let cancelled = Arc::new(AtomicBool::new(false));
        self.operations
            .lock()
            .expect("operation registry lock poisoned")
            .insert(operation_id.clone(), cancelled.clone());
        OperationToken { operation_id, cancelled }
    }

    pub fn cancel(&self, operation_id: &str) -> bool {
        let Some(flag) = self
            .operations
            .lock()
            .expect("operation registry lock poisoned")
            .get(operation_id)
            .cloned()
        else {
            return false;
        };
        flag.store(true, std::sync::atomic::Ordering::Relaxed);
        true
    }

    pub fn finish(&self, operation_id: &str) {
        self.operations
            .lock()
            .expect("operation registry lock poisoned")
            .remove(operation_id);
    }
}
```

Modify `src-tauri/src/lib.rs` to expose `pub mod v2;`, manage `OperationRegistry`, and register `cancel_operation`.

use std::path::Path;

#[derive(Debug, thiserror::Error)]
pub enum RecycleBinError {
    #[error("{0}")]
    Failed(String),
}

pub trait RecycleBin {
    fn move_to_recycle_bin(&self, path: &Path) -> Result<(), RecycleBinError>;
}

pub struct SystemRecycleBin;

impl RecycleBin for SystemRecycleBin {
    fn move_to_recycle_bin(&self, path: &Path) -> Result<(), RecycleBinError> {

    }
}

- [ ] **Step 3: Implement scanning**

Rules:

- Enumerate local drive roots from `selected_drives` and absolute `custom_folders`.
- Skip unreadable locations and count them in `skipped_locations`.
- Skip Windows, Program Files, Program Files (x86), ProgramData and protected paths for auto-selection.
- Filter by selected file types and sanitized custom extensions.
- Group candidates by file size first.
- Hash only size groups with at least two files.
- Use SHA-256 and expose only generated `hash_fingerprint_id`; do not persist the raw hash.
- Strict groups require same size and same hash.
- Suspected groups use same normalized stem and size delta within 2%; never auto-select suspected files.
